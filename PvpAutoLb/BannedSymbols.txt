# --- Threading: untracked threads leak into the game process on plugin unload ---
M:System.Threading.Thread.#ctor(System.Threading.ThreadStart);Use Svc.Framework.RunOnFrameworkThread or a tracked CancellationTokenSource — bare threads outlive the plugin and crash the game.
M:System.Threading.Thread.#ctor(System.Threading.ParameterizedThreadStart);Use Svc.Framework.RunOnFrameworkThread or a tracked CancellationTokenSource — bare threads outlive the plugin and crash the game.

# --- Task.Run / threadpool work: drops you off the framework thread; Dalamud APIs require it ---
M:System.Threading.Tasks.Task.Run(System.Action);Most Dalamud APIs require the framework thread — use Svc.Framework.RunOnFrameworkThread instead.
M:System.Threading.Tasks.Task.Run``1(System.Func{``0});Most Dalamud APIs require the framework thread — use Svc.Framework.RunOnFrameworkThread instead.
M:System.Threading.Tasks.Task.Run(System.Func{System.Threading.Tasks.Task});Most Dalamud APIs require the framework thread — use Svc.Framework.RunOnFrameworkThread instead.
M:System.Threading.Tasks.TaskFactory.StartNew(System.Action);Same issue as Task.Run — use Svc.Framework.RunOnFrameworkThread.

# --- Synchronous blocking on the framework thread freezes the game ---
M:System.Threading.Thread.Sleep(System.Int32);Sleeping the framework thread freezes the game — gate behavior with EzThrottler or DateTime checks instead.
M:System.Threading.Thread.Sleep(System.TimeSpan);Sleeping the framework thread freezes the game — gate behavior with EzThrottler or DateTime checks instead.
M:System.Threading.Tasks.Task.Wait;Sync wait on the framework thread deadlocks Dalamud — use a continuation or schedule on the next tick.
M:System.Threading.Tasks.Task.WaitAll(System.Threading.Tasks.Task[]);Sync wait deadlocks the framework thread — schedule continuations instead.
